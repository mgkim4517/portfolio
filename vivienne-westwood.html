Review

✓ 아쉬움

  • 둘이서 퍼블리싱을 완료해야 했기 때문에 시간이 촉박했습니다. 그래서 서로의 코드 리뷰 시간을 충분히 갖지 못했던 부분이 아쉬웠습니다.
  • 변수를 이용해 작업하지 못했던 부분이 아쉬웠습니다.

✓ 성과

  • 브랜드의 아이덴티티인 자유로움을 몽환적이고 자연 친화적인 느낌으로 디자인하고, 이를 기반으로 웹사이트를 구현했습니다.
  • MediaQuery를 이용해 모든 디바이스에서 대응 가능한 반응형 웹사이트를 구현했습니다.
  • 다양한 이벤트 구현을 통해 javaScript와 jQuery 능력을 향상시켰습니다.
  • 필요한 기술을 스스로 습득하여 다양한 이벤트를 구현했습니다.
  • ajax를 이용한 json 작업을 통해 유지보수가 쉽도록 DB를 관리했습니다.

✓ 배운 점

처음으로 반응형 웹사이트를 직접 기획부터 디자인, 퍼블리싱까지 해 보았습니다. 이 과정을 통해 퍼블리싱까지 고려해 디자인하는 방법에 대해 배웠습니다.

또한, GitHub Organization을 통해 다른 사람과 함께 퍼블리싱 작업하는 방법에 대해 배웠습니다. 다른 사람과 함께 진행하는 작업인 만큼 퍼블리싱 가이드가 중요하다는 것을 알게 되었습니다.

마지막으로, ajax를 이용한 json 작업을 해 보았습니다. 상품을 html에서 직접 넣는 것보다 유지보수가 훨씬 편하다고 느꼈습니다. 앞으로도 ajax를 이용한 json 작업을 통해 DB를 관리하고 싶습니다.

퍼블리싱 팀 프로젝트는 처음이라 어떻게 진행해야 할지 막막했습니다. 하지만 역할 분담 후 각자 맡은 역할을 충실히 수행하여 결과물을 완성할 수 있었습니다. 처음이라서 조금 아쉬움이 남는 프로젝트였습니다. 하지만 이번 프로젝트에서 쌓은 경험을 바탕으로 다음에 또 협업 기회가 생긴다면 더 나은 결과물을 만들고 싶습니다.
